# Lot Tracking Service (MES Portfolio Project)

A small backend service for tracking wafer lots moving through a
semiconductor fab, inspired by real MES (Manufacturing Execution
System) concepts from 300mm wafer fab operations.

## Domain model

A **lot** (a batch of wafers) moves through a sequence of **process
steps** (e.g. lithography, etch, deposition, inspection). At each
step it runs on a piece of **equipment** - note that step and
equipment are independent: the same step can run on more than one
tool, and the same tool can sometimes service more than one step.
Separately from step/equipment, a lot has a **state**: `WAITING`,
`PROCESSING`, `HOLD` (with a reason code), or - terminally -
`SCRAPPED` / `COMPLETED`.

Every change to a lot's (step, equipment, state) is recorded as an
append-only event, the same way a real MES keeps a transaction log
rather than overwriting history.

Built as a hands-on portfolio piece to demonstrate backend service
design (REST API, relational schema, containerization, observability,
cloud deployment) applied to a domain I have real production
experience in - MES / wafer fab operations - rather than a generic
tutorial app.

## API endpoints

| Method | Path                              | Purpose                                  |
|--------|-----------------------------------|-------------------------------------------|
| GET    | `/health`                         | Liveness check                            |
| POST   | `/lots`                           | Create a new lot                          |
| GET    | `/lots/{lot_id}`                  | Current snapshot (step, eqp, state, stuck)|
| POST   | `/lots/{lot_id}/events`           | Record a step/equipment/state change      |
| GET    | `/lots/{lot_id}/breakdown/by-step`     | Total time spent at each process step |
| GET    | `/lots/{lot_id}/breakdown/by-equipment`| Total time spent on each tool         |
| GET    | `/lots/{lot_id}/breakdown/by-state`    | Total time spent in each state        |
| GET    | `/lots/{lot_id}/holds`            | Count of each hold reason                 |

## Running locally

**1. Set up MySQL** (assumes a MySQL server is already running locally):

```sql
CREATE DATABASE lot_tracking;
CREATE USER 'lot_app'@'localhost' IDENTIFIED BY 'choose-a-password';
GRANT ALL PRIVILEGES ON lot_tracking.* TO 'lot_app'@'localhost';
FLUSH PRIVILEGES;
```

Then create the schema:

```sql
USE lot_tracking;

CREATE TABLE lots (
    lot_id VARCHAR(50) PRIMARY KEY,
    wafer_count INT NOT NULL,
    current_step VARCHAR(20) NOT NULL,
    current_eqp_id VARCHAR(50) NOT NULL,
    current_state VARCHAR(20) NOT NULL
);

CREATE TABLE lot_events (
    event_id INT AUTO_INCREMENT PRIMARY KEY,
    lot_id VARCHAR(50) NOT NULL,
    step_id VARCHAR(20) NOT NULL,
    eqp_id VARCHAR(50) NOT NULL,
    state VARCHAR(20) NOT NULL,
    hold_reason VARCHAR(30) NULL,
    timestamp DATETIME NOT NULL,
    FOREIGN KEY (lot_id) REFERENCES lots(lot_id)
);
```

**2. Create a `.env` file** in the project root (never committed - see `.gitignore`):

```
DB_HOST=localhost
DB_USER=lot_app
DB_PASSWORD=choose-a-password
DB_NAME=lot_tracking
```

**3. Install dependencies and run:**

```bash
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Then visit `http://localhost:8000/docs` for interactive API docs
(auto-generated by FastAPI from the Pydantic schemas in `app/schemas.py`).

Data now persists in MySQL across server restarts. The original
in-memory implementation is kept at `app/store_inmemory.py` for
reference - both expose the identical `Store` interface, demonstrating
the repository pattern in practice.

## Running in Docker

Build the image:

```bash
docker build -t lot-tracking-service .
```

Run it, pointing the container at MySQL running on the host machine
(Docker Desktop on Mac/Windows exposes the host as `host.docker.internal`;
the `.env` file's own `DB_HOST=localhost` is only correct when running
uvicorn directly, not inside a container):

```bash
docker run -d \
  --name lot-tracker \
  -p 8000:8000 \
  --env-file .env \
  -e DB_HOST=host.docker.internal \
  lot-tracking-service
```

Check it's running and view logs:

```bash
docker ps
docker logs lot-tracker
```

Same `http://localhost:8000/docs` works identically whether the app
is running directly via uvicorn or inside this container.
